⚡ BitFabric Quickstart

Get started with pub/sub messaging in minutes

Installation

BitFabric is designed for zero-friction integration. Add it to your project with one command:

npm install bitfabric

Instant Demo

Want to see it in action without coding? Open our Web Console and hit Connect.

Instant P2P Messaging

1. Initialize (Zero Config)

Create a fabric instance using the global Free Tier. No registration required.

import { PubSubFabric } from 'bitfabric';

const fabric = new PubSubFabric({
  roomId: 'bitfabric-free-tier'
});

await fabric.init();

2. Subscribe to a Topic

Listen for real-time messages on any channel:

fabric.subscribe('general-support', (message) => {
  console.log(`From ${message.from}: ${message.data.text}`);
});

3. Publish a Message

Broadcast data across the global mesh instantly:

fabric.publish('general-support', { 
  text: 'Hello from my decentralized app!'
});

Going Pro: Custom Rooms

When you're ready for dedicated namespaces and custom topics, upgrade to a Validated Session.

  • Step 1: Sign up at bitfabric.cc to get your API Key.
  • Step 2: Use your key to unlock custom roomId namespaces.
  • Step 3: Publish to restricted topics without mapping limits.
Note: Free Tier users are automatically remapped to bitfabric-free-tier to ensure network stability for everyone.

Chat Demo Example

Complete working example:

import { createChatDemo } from 'bitfabric/demo';

// Create a chat room
const chat = await createChatDemo('room-name');

// Send messages
chat.sendMessage('Hello everyone!');

// View stats
console.log(chat.getStats());
// { messagesSent: 1, messagesReceived: 0, peersOnline: 1 }

// Cleanup
await chat.destroy();

Subscribe to Multiple Topics

Listen to several channels at once:

const topics = ['chat', 'notifications', 'system'];

topics.forEach(topic => {
  fabric.subscribe(topic, (message) => {
    console.log(`[${topic}] from ${message.from}:`, message.data);
  });
});

Advanced Configuration

BitFabric automatically detects and connects to global Gun and Nostr relays, but you can provide your own for dedicated infrastructure.

const fabric = new PubSubFabric({
  roomId: 'your-room-id',
  nostrRelays: [
    'wss://nos.lol',
    'wss://relay.damus.io'
  ],
  gunPeers: [
    'https://relay.peer.ooo/gun'
  ]
});
Note: gunPeers and nostrRelays are optional. The library will auto-detect optimal relays if these lists are omitted.

Publish with Metadata

Send structured data:

fabric.publish('data', {
  type: 'sensor',
  value: 42,
  location: 'living-room',
  timestamp: Date.now()
});

How It Works

BitFabric uses a dual-transport pub/sub architecture:

Component Role
Gun Primary distributed database sync
Nostr Secondary relay layer for reach
UnSEA End-to-end encryption for all messages
No WebRTC Direct relay-based messaging (simpler, faster)
Message Flow: Client → Gun Relay (encrypted) → Subscribers + Nostr Relay (encrypted) → Additional reach

Configuration

Customize your fabric instance:

const fabric = new PubSubFabric({
  roomId: 'required-room-id',           // Room identifier
  nostrRelays: [                        // Optional relay list
    'wss://nos.lol',
    'wss://relay.damus.io'
  ]
});

Error Handling

Handle connection errors gracefully:

try {
  await fabric.init();
} catch (err) {
  console.error('Failed to initialize fabric:', err);
}

fabric.on('error', (error) => {
  console.error('Transport error:', error);
});

Performance Tips

  • Batch messages - Group related publishes together
  • Limit subscriptions - Only subscribe to topics you need
  • Cache topics - Reuse fabric instances for same room
  • Monitor stats - Use getStats() to track performance
