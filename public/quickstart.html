⚡ BitFabric Quickstart

Get started with pub/sub messaging in minutes

Installation

Clone and run the development server:

npm install
npm run dev
# Open http://localhost:5173

Install as a package

If/when BitFabric is published to npm, install it like this:

npm install bitfabric

Basic Usage

1. Initialize the Fabric

Create a new pub/sub fabric instance:

import { PubSubFabric } from './src/fabric/index.js';

const fabric = new PubSubFabric({
  roomId: 'my-chat-room',
  nostrRelays: ['wss://relay.damus.io', 'wss://relay.nostr.band']
});

await fabric.init();

2. Subscribe to a Topic

Listen for messages on a topic:

fabric.subscribe('messages', (message) => {
  console.log(`${message.from}: ${message.data.text}`);
});

3. Publish a Message

Send data to all subscribers:

fabric.publish('messages', { 
  text: 'Hello from BitFabric!',
  timestamp: Date.now()
});

4. Unsubscribe

Stop listening to a topic:

const unsubscribe = fabric.subscribe('events', handler);
// Later...
unsubscribe();

Chat Demo Example

Complete working example:

import { createChatDemo } from './src/fabric/demo.js';

// Create a chat room
const chat = await createChatDemo('room-name');

// Send messages
chat.sendMessage('Hello everyone!');

// View stats
console.log(chat.getStats());
// { messagesSent: 1, messagesReceived: 0, peersOnline: 1 }

// Cleanup
await chat.destroy();

Subscribe to Multiple Topics

Listen to several channels at once:

const topics = ['chat', 'notifications', 'system'];

topics.forEach(topic => {
  fabric.subscribe(topic, (message) => {
    console.log(`[${topic}] from ${message.from}:`, message.data);
  });
});

Publish with Metadata

Send structured data:

fabric.publish('data', {
  type: 'sensor',
  value: 42,
  location: 'living-room',
  timestamp: Date.now()
});

How It Works

BitFabric uses a dual-transport pub/sub architecture:

Component Role
Gun Primary distributed database sync
Nostr Secondary relay layer for reach
UnSEA End-to-end encryption for all messages
No WebRTC Direct relay-based messaging (simpler, faster)
Message Flow: Client → Gun Relay (encrypted) → Subscribers + Nostr Relay (encrypted) → Additional reach

Configuration

Customize your fabric instance:

const fabric = new PubSubFabric({
  roomId: 'required-room-id',           // Room identifier
  nostrRelays: [                        // Optional relay list
    'wss://nos.lol',
    'wss://relay.damus.io'
  ]
});

Error Handling

Handle connection errors gracefully:

try {
  await fabric.init();
} catch (err) {
  console.error('Failed to initialize fabric:', err);
}

fabric.on('error', (error) => {
  console.error('Transport error:', error);
});

Performance Tips

  • Batch messages - Group related publishes together
  • Limit subscriptions - Only subscribe to topics you need
  • Cache topics - Reuse fabric instances for same room
  • Monitor stats - Use getStats() to track performance
